# 🧠 戦車ゲーム開発を通じて学ぶJAVAとオブジェクト指向プログラミング

## 🎮 概要

- 就業支援訓練に平行して、演習としてターン制の戦車バトルゲーム（標準24×24マップ..可変）を作成する
- JAVAの学習進捗に応じて内容をアップデート。最終的にはPytionを使った機械学習で敵戦車の戦闘力を高めるまでを目標とする
- JAVAの開発は現状では講義で使うeclipse（2024)
- AIを積極的に活用し、最終的には実際のコーディングはほぼAIで、それを読んで修正、または意図したコードになるように仕様を修正するような開発スタイルを確立する
- ただし、AIの活用は無料サービスの範囲にとどめる

## 🎮 GitHubに登録する前の経過

- オブジェクト指向の学習段階に入った段階で、演習として人vsコンピュータの戦車戦ゲームを作ることを発案 
  - 基本戦車クラスを作成。さらに、継承によって、性能の異なる機種クラスを作成。
  - インスタンス生成で複数の戦車を作り、動かすメインシーケンスを作成。java.util.Scanner とコンソール画面によるCUI版完成（2月初旬） 
- AIにコードを読ませ、ゲームのGUI化を指示。以後グラフィックはAIコードをそのまま使用
  - 主通りのものができたのと、グラフィックの学習は目的外(GUIの本命はWeb）と捉えている為 
- 敵戦車の行動をより賢いものにするため、（言葉だけしか知らない）機械学習の導入を検討し、調査開始
  - AIでのリサーチや相談を続けるにあたり、内容を整理し保存する先としてNotionの利用を開始
- 機械学習にも対応できるように敵戦車AIの改修とクラス分離を実施。
  - Notion上で計画概要や仕様素案などを作成　→　AIへレビュー　→　AIと壁打ちしてモディファイ →　まとめをNotionに記録（AI)  というワークフローを確立
- 今後の機能拡張にそなえ、既存クラスに加え、AIに任せる描画、プレーヤー戦車を動かす制御などもクラス分割。
  - その時期にGitHubのことを知り、AIに相談してこのプロジェクトをアップすることになった
  - 上記のように学んだことをすぐ試すところから始めており、これまでの開発は無計画なので、今後は整理も平行しながら進めていく
  - AIを含め、初めての経験ばかりで失敗も多く、上書き等で初期のソースや仕様書の多くが失われてしまった。
  
## 今後の計画と進捗状況

現時点で、とりあえず遊べるゲームとしては完成しているので、今後は以下のように進めることとした。  

🧮 フェーズ1：スコアリングAI 　（初版完了）
状況や行動の価値を数値し、それらを評価計算して、その結果で行動を選択する敵戦車AIを作る  
3/3 EnemyAI2　初版完成  いくつが不具合が残るがそれを修正したら次の段階に進む。

📊 フェーズ2：戦闘データを記録する　←**先にフェーズ３を進め、そのつぎここ**
- 簡単なログを出すようにして、アルゴリズムのマニュアルチューニングを実施  
  （決着がつかない、とか、対戦が成立しないような明らかな不具合を修正）**※3/4済**
- いったんこの状態で自己対戦を実施するためフェーズ3に進む  
  今後は機械学習に必要なデータが何かを調査し、そのログを採取する。　

AIからの項目提案はあるので、それを今後検証予定。

🤖 フェーズ3：自己対戦シミュレーション　←**今ここ**
- プレーヤー行動のクラス化しAIとの差し替え可能化　**※3/4済**
- フェーズ1のアルゴリズムプレーヤー側AIを作成    **※3/4済**
- GUIを削除した高速化バージョンをAIを使って作成　**※3/6済**
- AI同士を数千回対戦  　                       **※3/6済**
  同条件では先手勝率が60%

🏆 フェーズ4：強化学習(開発休止）
ログをもとに強化学習（詳細はPython講義開始後に検討）を検討したが、以下の理由で休止する
何度も戦闘を行った経験から、現状のパラメータの勝敗に対する感度が鈍く、最適化学習では強化の効果が見えそうにないことが分かった。
学習によって強化できる戦闘アルゴリズムについて、それなりの研究が必要だが、それだとプログラム学習の目的からかなり離れてしまう。
当初の目的であるオブジェクト指向プログラミンの演習については十分に学べたので、このプログラムの開発はここでいったん終了とする
